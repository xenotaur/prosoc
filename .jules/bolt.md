## 2024-05-18 - Caching JSON Schema Validation in Python
**Learning:** `jsonschema.validate` compiles the schema on every call, which is expensive. Reusing a compiled `Validator` instance (e.g., `Draft7Validator(schema).validate(instance)`) provides an enormous speedup. For `jsonschema.validate(instance, schema)` called in a loop (like compiler validation or runtime data checking), caching the compiled validator object transforms O(n) validation time back to near-constant overhead.
**Action:** Always prefer instantiating a validator like `Validator = jsonschema.validators.validator_for(schema); validator = Validator(schema); validator.validate(instance)` or `Draft7Validator(schema).validate(instance)` over using the top-level `jsonschema.validate` function, especially if validation happens multiple times or on hot paths.
